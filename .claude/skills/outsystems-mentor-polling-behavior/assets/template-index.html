What is outsystems-mentor-polling-behavior skill?

Run Mentor efficiently. This skill handles the polling discipline so you don't have to.

When you ask Mentor to do something, it runs asynchronously — the agent has to keep checking "are you done yet?" every few milliseconds. By default, that check fires every 500ms. On a 6-minute Mentor run, that's over 700 checks — each one bringing verbose intermediate output back into the conversation and burning tokens. This skill changes the cadence to 30 seconds, reducing those 700 checks to about 12, with no change to how long Mentor takes to finish.

It also records what happened — poll count, timing, how much context was saved — so you can benchmark over time and know if the discipline is working.

Three things it does:

  • Say "use Mentor on [app] to do X" — runs Mentor with correct polling discipline
  • Say "generate the Mentor dashboard" — renders your session history as HTML
  • Say "Mentor session summary" — cross-session digest and trend analysis

One thing it doesn't do: it doesn't make Mentor faster. The run takes the same time. You just stop paying for the polling overhead.
