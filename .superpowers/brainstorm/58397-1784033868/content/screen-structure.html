How many screens?

Standard ODC CRUD: 2-screen vs 3-screen pattern

● CustomerList
[ New Customer ]
┌──────────────────────────────────┐
│ CUSTOMER_ID │ NAME │ EMAIL │ ··· │
│ CID-001 │ John Smith │ j@… │ ✏️ 🗑️ │
│ CID-002 │ Jane Doe │ d@… │ ✏️ 🗑️ │
└──────────────────────────────────┘
● CustomerDetail (input: CustomerId?)
Title: "New Customer" OR "Edit Customer"
[ CUSTOMER_ID ] [ FIRST_NAME ]
[ LAST_NAME ] [ EMAIL ]
[ CREATED_AT — read only ]
[ Save ] [ Cancel ]

A — 2 Screens

CustomerList + CustomerDetail (shared create/edit). ODC standard pattern. CustomerId empty = create, populated = edit.

Pros

  • Less to build
  • One place for form logic

Cons

  • Slightly more conditional logic in one screen
● CustomerList
[ New Customer ] → CustomerCreate
┌──────────────────────────────────┐
│ CUSTOMER_ID │ NAME │ EMAIL │ ··· │
│ CID-001 │ John Smith │ j@… │ ✏️ 🗑️ │
└──────────────────────────────────┘
● CustomerCreate
Blank form → calls Customer_Create
● CustomerEdit (input: CustomerId)
Pre-filled form → calls Customer_Update

B — 3 Screens

CustomerList + CustomerCreate + CustomerEdit. Separate screens, cleaner logic per screen.

Pros

  • Simpler logic per screen
  • Titles always correct

Cons

  • More screens to build
  • Duplicate form layout
