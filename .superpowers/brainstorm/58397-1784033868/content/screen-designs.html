Key behaviours

  • CustomerList OnInitialize → Customer_GetAll → bind to table
  • Delete button → confirm dialog → Customer_Delete(CUSTOMER_ID) → refresh list
  • Edit button → navigate to CustomerDetail with CustomerId
  • New Customer → navigate to CustomerDetail with no CustomerId
  • CustomerDetail OnInitialize → if CustomerId not empty → Customer_GetById → populate local variable
  • Save (create) → Customer_Create → if Success → navigate to CustomerList
  • Save (edit) → Customer_Update → if Success → navigate to CustomerList
  • Errors → shown via ODC feedback message on the screen
  • CUSTOMER_ID → editable only in create mode, disabled in edit mode
  • CREATED_AT → shown read-only in edit mode, hidden in create mode
